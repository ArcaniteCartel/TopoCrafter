## Measure Bar Tick Positioning Bug Fix

### Bug

When framing was enabled with large bottom or left margins (e.g. 80 px), enabling measure bars caused the bottom tick bar to render inside the map image area rather than in the bottom margin area.

### Root Cause

`MeasureBarOverlay` received `outerW` and `outerH` from a `useState` value (`outerSize`) that was populated by a `ResizeObserver` on the outer composition div. `ResizeObserver` defaults to observing the **content-box** of its target. When the user changes a frame margin (which is CSS `padding` on the outer div), the content area of the outer div does **not** change — only the padding does — so the ResizeObserver never fires and `outerSize` stays permanently stale with whatever values it had when the image first loaded.

With a stale `outerH` that was computed with the old (smaller) bottom margin, the formula `y0 = outerH - mb` used the new (larger) `mb` against the old `outerH`, placing `y0` inside the map area rather than at its bottom edge.

### Fix (`src/renderer/src/components/MapCanvas/MapCanvas.tsx`)

Moved the ResizeObserver from the outer composition div to the **inner map div**. The inner map div's content-box changes correctly when the image loads or zoom changes (the original triggers), so the ResizeObserver still fires in all meaningful cases.

Frame margins are now added to live React state values at render time inside `MeasureBarOverlay` rather than being baked into the observed size:

```typescript
// Before — MeasureBarOverlay received pre-computed outer dimensions (stale on margin change):
outerW={outerSize.w}
outerH={outerSize.h}
// ...inside MeasureBarOverlay:
const mapW = outerW - ml - mr
const mapH = outerH - mt - mb

// After — MeasureBarOverlay receives inner map area dimensions (always current via ResizeObserver):
mapW={innerMapSize.w}
mapH={innerMapSize.h}
// ...inside MeasureBarOverlay:
const outerW = mapW + ml + mr   // ml/mr are live from frame state
const outerH = mapH + mt + mb   // mt/mb are live from frame state
```

**Specific changes:**

- Renamed `outerDivRef` → `innerMapRef`; removed `ref` from the outer composition div; added `ref={innerMapRef}` to the inner map div.
- Renamed `outerSize/setOuterSize` → `innerMapSize/setInnerMapSize`; ResizeObserver now observes `innerMapRef.current`.
- `MeasureBarOverlay` props changed from `outerW: number, outerH: number` to `mapW: number, mapH: number`.
- Inside `MeasureBarOverlay`, the two lines computing `mapW/mapH` from `outerW/outerH` replaced by two lines computing `outerW/outerH` from `mapW/mapH + frame.margin*`.
- Render condition updated: `outerSize &&` → `innerMapSize &&`.

TypeScript clean (`npx tsc --noEmit` no output).

## Terrain Ruggedness Index (TRI) Tool

### Feature Overview

Added a new **Terrain Ruggedness Index** tool — the second button in the tools popover (left of the elevation flag tool). Hovering over the map shows a live TRI value; clicking drops a persistent ruggedness flag marker. Full drag and delete support, same as elevation flags and slope arrows.

**TRI formula**: Riley et al. 1999 — `sqrt(Σ(center − neighbor)²)` for 8 neighbors, computed in normalized heightmap space (`0–1`).

**Display value**: `triNorm × |realMax − realMin|` in calibration units when elevation range is known; raw `triNorm.toFixed(3)` (dimensionless) otherwise.

**Severity tiers** (5 levels, used for both flag color and legend color bar):

| Tier | triNorm range | Color |
|---|---|---|
| Very Low | < 0.004 | `#4CAF50` (green) |
| Low | 0.004–0.015 | `#8BC34A` (yellow-green) |
| Moderate | 0.015–0.04 | `#FFC107` (amber) |
| High | 0.04–0.1 | `#FF5722` (deep orange) |
| Extreme | > 0.1 | `#8B0000` (dark red) |

### Flag Icon Design

Vertical pole + two jagged/sawtooth polylines going upper-right from the pole top:
- Upper ridge: steep peaks (mountain silhouette)
- Lower ridge: shallow undulation (foothills)

Toolbar icon: same geometry in 16×16 SVG (`RuggednessIcon` component).

### Color-by-Severity Toggle

New **Markers & Annotations** collapsible section in ParameterPanel contains:
- `Switch` — "Color ruggedness flags by severity" (default: on). When on, each flag uses its severity tier color; when off, all flags use the contour style `labelColor`.

### Legend Integration

When ruggedness flags are present and `legend.showRuggednessFlags` is enabled:
- A **ruggedness** icon item appears in the normal legend items grid (with configurable label, default `'Ruggedness index'`).
- A full-width **TRI color bar** is appended below the legend items grid — 5 colored rectangles with tier labels underneath.
- Both the SVG legend overlay and the canvas export legend match.
