### ParameterPanel — Framing section

Added a collapsible "Framing" section (collapsed/expanded state: `framingOpen`) above the
Contour Parameters divider. Controls:

- **Enable frame** — master `Switch`; all sub-controls disabled when off
- **Margin color** — `ColorInput`
- **Margin (px)** — four `NumberInput` fields (Top, Bottom, Left, Right), step 5, max 500
- **Show border** — `Switch`; border sub-controls disabled when off
- **Border color** — `ColorInput`
- **Border width (px)** — `NumberInput`, step 0.5, max 20
- **Border style** — vertical `SegmentedControl` with five options:
  Single line / Double line / Cartographic / Drop shadow / Ornate

### MapCanvas — live frame preview

The inner zoom div was split into two layers:

1. **Outer composition div** — gets CSS `padding` (the four margins) and `backgroundColor`
   (the margin color) when `frame.enabled && heightmap`. Width still set by `mapZoom%`.

2. **Inner map area div** — position relative, contains the base image and all SVG overlays
   (contour + annotation). Unchanged behaviour; `overlayOnly` background still applies here.

3. **`FrameBorderOverlay` component** — absolutely positioned over the full outer composition
   (margins + map), using CSS div borders. No SVG calc() issues. Per-style rendering:

| Style | CSS technique |
|-------|--------------|
| `single` | `border` |
| `double` | `border` + `outline` with negative `outlineOffset` |
| `cartographic` | solid outer `border` + dashed inner absolutely-positioned div |
| `shadow` | `border` + `box-shadow` |
| `ornate` | double `border`/`div` + four filled corner squares at inner border corners |
