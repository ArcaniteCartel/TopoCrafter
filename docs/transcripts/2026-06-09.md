## ParameterPanel — GEOLOCATION Top-Level Group

### Change

Geolocation parameters moved out of Framing › measure bars into a new independent top-level group **GEOLOCATION**, inserted between BUILDINGS and GRIDS.

### What Moved

| Item | From | To |
|------|------|----|
| Anchor latitude / longitude | Framing › measure bars › geo block | GEOLOCATION group |
| Planet radius | Framing › measure bars › geo block | GEOLOCATION group |
| Precision setting | Framing › measure bars › geo block | Calibration section (alongside map width) |

### What Stayed in Measure Bars

- **Show geo coordinates** toggle — now disabled unless `hasGeoInfo` is true (map width set AND any anchor value differs from the 0/0 defaults). Hover tooltip when disabled: "Configure geolocation settings in the Geolocation group to enable this control."
- **Horizontal axis = latitude** toggle — shown when `geoEnabled`

### Design Decisions

| Decision | Choice |
|----------|--------|
| GEOLOCATION gating | No frame/measureBar gating — always independently configurable |
| "Show geo coordinates" disable condition | `!hasGeoInfo` (replaces old `!frame.enabled \|\| !measureBar.enabled \|\| !hasGroundResolution`) |
| `hasGeoInfo` definition | `hasGroundResolution && (anchorLat !== 0 \|\| anchorLon !== 0 \|\| anchorX !== null \|\| anchorY !== null)` |
| Precision setting location | Calibration, shown when `sagittalErrorM !== null` |
| `effectivePrecision` formula | Removed `measureBar.enabled &&` — now just `measureBar.geoEnabled ? precisionSetting : 'medium'` |
