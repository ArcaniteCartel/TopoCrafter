## Fix: Real-World Contour Interval Field Not Editable

**Problem:** The "Interval (ft)" TextInput in the Parameter Panel accepted focus but blocked all keyboard input.

**Root cause:** The field had `disabled={!calReady}` ŌĆö Mantine's disabled input blocks the underlying `<input>` element even though the wrapper div appears focusable.

**Fix:** Removed the `disabled` prop entirely from the TextInput. Added a `calReady` guard inside `handleIntervalInput` so store updates are skipped when calibration isn't ready, but the user can still type freely.

**File:** `src/renderer/src/components/ParameterPanel/ParameterPanel.tsx`

## Architecture Change: Decouple Controls from Recalculation

**Problem:** Every parameter change auto-triggered contour recalculation (via `useMemo`) or hillshade regeneration (via a 300ms debounced `useEffect`), causing performance issues and interfering with user input.

**Design decision (Q&A with user):**
- Style controls (colors, widths, opacity, label toggle) remain live/instant ŌĆö no recalc needed
- Hillshade controls + contour controls require explicit user action to apply
- "Rerun Hillshading" button hidden when not in hillshade mode
- Fresh heightmap load still auto-runs the full pipeline

**Implementation:**

### Store additions (`src/renderer/src/store/useStore.ts`, `src/renderer/src/types/index.ts`)
- Added `hillshadeDirty: boolean`, `contoursDirty: boolean`, `hillshadeVersion: number`, `contoursVersion: number` to `ProjectState`
- `updateHillshadeParams` now sets `hillshadeDirty: true`
- `updateParameters`, `updateElevationCalibration`, `setElevationUnits` now set `contoursDirty: true`
- `setTerrainIsHillshade(true)` increments `hillshadeVersion` (triggers auto-run on load)
- `setHeightmap` increments `contoursVersion` (triggers auto-run on load)
- New actions: `triggerHillshade()` (increments version, clears dirty), `triggerContours()` (same), `setHillshadeDirty()`, `setContoursDirty()`

### Hook: `src/renderer/src/hooks/useHillshade.ts`
- Removed `hillshadeParams` from `useEffect` deps ŌĆö no longer auto-fires on param changes
- Added `hillshadeVersion` to deps ŌĆö fires only when explicitly triggered or on load
- Added `hillshadeParamsRef` so the async callback always reads latest params without stale closure

### MapCanvas: `src/renderer/src/components/MapCanvas/MapCanvas.tsx`
- Replaced `useMemo` contour generation with `useState` + `useEffect([heightmap, contoursVersion])`
- Added `parametersRef` so the effect always reads the latest parameters when triggered
- Contours only recompute on new heightmap load or explicit "Recalculate Contours" click

### FilePanel: `src/renderer/src/components/FilePanel/FilePanel.tsx`
- Added two new Paper-bordered button sections below the existing load buttons, styled identically:
  - **"Rerun Hillshading"** ŌĆö visible only in hillshade mode; enabled when `hillshadeDirty`; shows loading spinner while generating; disables after run
  - **"Recalculate Contours"** ŌĆö always visible; enabled when `contoursDirty && heightmap`; synchronous, no spinner needed
- After heightmap load: explicitly calls `setHillshadeDirty(false)` and `setContoursDirty(false)` to keep buttons dormant after the auto-triggered run

## Fix: Contour Label Values Incorrect

**Problem:** With min=0 ft, max=1500 ft, interval=50 ft, the first major contour (every 5th) was labeled 77 instead of 250.

**Root cause:** Label formula treated the normalized threshold as a 0ŌåÆ1 fraction of the full range:
```ts
// Wrong
const elev = Math.round(realMin + threshold * (realMax - realMin))
```
But thresholds run from `parameters.minElevation` to `parameters.maxElevation` (e.g., 0ŌåÆ0.3063), not 0ŌåÆ1. So `threshold=0.05105` produced `0.05105 ├Ś 1500 = 76.5 Ōēł 77` instead of the correct 250.

**Fix:** Correct linear interpolation using the actual normalized span:
```ts
const elev = Math.round(realMin + (threshold - normMin) / normSpan * (realMax - realMin))
```

**File:** `src/renderer/src/components/MapCanvas/MapCanvas.tsx`

## Fix: Label Off-By-One Rounding (e.g., 749 instead of 750)

**Problem:** With interval=75 ft, major contour labels showed 749, 1124, 1499 instead of 750, 1125, 1500.

**Root cause:** Floating-point accumulation in the threshold array (generated by repeated `+= interval` in a loop) caused values like threshold[10] = `0.149999...` instead of exactly `0.15`. Dividing by `normSpan` and multiplying by `realSpan` then landed just below the integer, rounding down.

**Fix:** Bypass the threshold entirely. Use the contour index `i` and the integer `realInterval` for exact arithmetic:
```ts
const elev = (realInterval !== null)
  ? realMin + i * realInterval          // exact integers always
  : Math.round(realMin + (threshold - normMin) / normSpan * (realMax - realMin))  // fallback
```
Since `realMin` (0) and `realInterval` (75) are both integers, `0 + i ├Ś 75` is always exact.

**File:** `src/renderer/src/components/MapCanvas/MapCanvas.tsx`
