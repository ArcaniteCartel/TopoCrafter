## Sagittal Error System

### Background

A flat rectangular map tile deviates from the spherical planetary surface. The maximum deviation (at the tile center) is given by:

```
D = √(W² + L²)          // diagonal in meters
s = R − √(R² − (D/2)²)  // exact sagittal error
s ≈ D² / (8R)            // small-angle approximation
```

Where W and L are real-world tile dimensions and R is the planet radius. For a 100 × 100 km tile on Earth, s ≈ 390 m — significant for precision work.

### New Files

**`src/renderer/src/utils/sagittal.ts`**

Pure utility module:
- `computeSagittalErrorM(mapWidthUnits, pixelW, pixelH, unitType, planetRadiusKm)` — returns `null` for custom units (no reliable meter conversion)
- `sagittalColor(errorM)` — Mantine color token: `green.7` ≤ 2 m, `yellow.7` ≤ 10 m, `orange.7` ≤ 30 m, `red.7` > 30 m
- `formatSagittalError(errorM)` — formats as `X.X m` or `X.XX km`
- `PRECISION_CAPS` — `{ high: 2, medium: 10, low: 30 }`
- `PRECISION_LABELS` — human-readable select labels

### Type Changes (`types/index.ts`)

```ts
export type PrecisionSetting = 'high' | 'medium' | 'low'
```

Added to `ProjectState`:
```ts
precisionSetting: PrecisionSetting       // default: 'medium'
sagittalExceptionAcknowledged: boolean   // default: false
```

### Store Changes (`store/useStore.ts`)

- Version bumped to `2`; migration backfills defaults for new fields
- New actions:
  - `setPrecisionSetting(s)` — sets precision, clears acknowledgment
  - `setSagittalExceptionAcknowledged(v)`
- `updateElevationCalibration` clears `sagittalExceptionAcknowledged` whenever `mapWidth` is included in the update (auto-revoke: if the user reduces the tile to bring error within cap, the old acknowledgment is stale)
- Both fields added to `partialize` and `merge`

### Design Decisions

| Decision | Choice | Reason |
|----------|--------|--------|
| Effective precision when geo not enabled | Always `'medium'` (10 m) | User specified: "defaults to 10 m error cap if area not activated" |
| Auto-revoke exception | Yes, on any `mapWidth` change | User confirmed: "if adequately reduced, auto-revoke" |
| `setPrecisionSetting` clears exception | Yes | Tightening the cap invalidates the old acknowledgment |
| Color bands | green ≤ 2 m / yellow ≤ 10 m / orange ≤ 30 m / red > 30 m | One red band (user confirmed: "one red band") |
| Rejection behavior | Soft — persistent orange Alert with checkbox | Hard input block is awkward in React controlled inputs |
| Precision selector location | Inside `geoEnabled` block in Framing › measure bars sub-subgroup | User: "put it in the geolocation area" |
| Sagittal indicator location | Toolbar, between display size and Export | User: "top bar is in main layout" |

### Toolbar Indicator

Always visible when `mapWidth` and `heightmap` are both set (regardless of unit type, as long as it's feet or meters):

- **Left element:** colored error value (e.g. `⊕ 12.3 m`) with a tooltip explaining the sagittal error concept and the color thresholds
- **Right element:** cap label (e.g. `/10 m`) with a tooltip: "Precision cap: 10 m. Change in Calibration › Geolocation."

Effective precision = `precisionSetting` when `measureBar.enabled && measureBar.geoEnabled`; otherwise always `'medium'`.

### ParameterPanel — Calibration Section

After the map width input, two new elements appear when a sagittal error can be computed:

1. **Inline display:** `Spherical error: X.X m / Y m cap` — the error value has a hover tooltip with a full explanation
2. **Warning Alert** (orange, appears only when `sagittalErrorM > effectiveCapM`):
   - Describes the error in plain language: tile size, deviation amount, which annotations are affected
   - Includes a `Checkbox` — "I understand the limitation and wish to proceed"
   - When checked: Alert turns gray (acknowledged state); all functionality remains available
   - When `mapWidth` changes: checkbox is automatically unchecked (`sagittalExceptionAcknowledged` reset)

### ParameterPanel — Geolocation Block

When `measureBar.geoEnabled && hasGroundResolution`, a `Select` appears at the top of the geolocation controls:

```
Precision setting
High — ≤ 2 m (local / tactical)
Medium — ≤ 10 m (balanced)        ← default
Low — ≤ 30 m (macro / regional)
```

When geolocation is not active, the effective cap is silently Medium with no selector shown.
