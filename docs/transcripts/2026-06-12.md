## Feature: Vegetation Layer Generation

### Overview

Multi-layer vegetation overlay driven by water proximity (lake shore spread + river corridor spread), with five texture styles, elevation thinning, and per-layer colour/opacity. Layers sit between the terrain/hillshade image and the contour SVG in the render stack.

### New File: `src/renderer/src/utils/vegetation.ts`

**Water mask rasterization:**
- `rasterizePolygon` — scanline fill of lake polygons into a `Uint8Array` mask
- `rasterizeLine` — Bresenham line + circular brush for river polylines (respects `strahlerOrder`-scaled thickness)

**BFS distance transform:**
- `bfsDistance(seedMask, w, h)` — 4-connected BFS flood from seed pixels, returns `Float32Array` of per-pixel distances; separate transforms for lakes and rivers

**Noise utilities (no external deps):**
- `valueNoise` / `hash2` — smooth hash-based value noise
- `fbm` — fractional Brownian motion (layered value noise, configurable octaves)
- `worley` — Worley/cellular F1 distance with configurable jitter
- `BAYER8` — normalized 8×8 Bayer dither matrix

**`generateVegetation(input): string` pipeline:**
1. Rasterize lake polygons + river polylines into separate water masks
2. BFS distance transforms for each mask
3. Per-pixel base density from exponential water-proximity falloff: `exp(-3 * dist / spreadPx)`
4. Texture modulation (5 styles — see below)
5. Blend with global `noisiness`-weighted fBm
6. Elevation thinning with per-pixel fBm variation on the cutoff + water-proximity attenuation
7. Render to offscreen canvas, return data URL

**5 texture styles:**
| Style | Description |
|-------|-------------|
| `gradient` | Pure exponential falloff, no texture |
| `organic` | fBm value noise, configurable octaves (1–8) |
| `stipple` | Bayer 8×8 dither threshold, modulated by fBm |
| `hatch` | Trigonometric diagonal lines at configurable angle + spacing |
| `cellular` | Worley F1 distance noise with configurable jitter |

**`VegetationGenInput` fields:**
- `pixelsPerUnit: number` — `1` when uncalibrated (spread stored as px), else `1 / (calUnitsPerPixel)` so display-unit spread values convert to pixels at generation time

## Bugs Fixed

### Bug 1: Spread values eaten while typing

**Problem:** Spread controls stored values in pixels (`lakeSpreadPx`) and converted on every `onChange` keystroke. With a calibrated map (e.g. 5 ft/px), typing "1" → stored as 0 px → displayed as 0 ft → "1" was discarded.

**Fix:** Renamed `lakeSpreadPx`/`riverSpreadPx` → `lakeSpread`/`riverSpread` in `VegetationLayer`. Controls display and store exactly what the user types (in their calibration unit or px). Conversion to pixels happens only inside `generateVegetation` via `pixelsPerUnit`. ParameterPanel computes `spreadPxToUnit = mapWidth / heightmap.width` and passes `pixelsPerUnit = 1 / spreadPxToUnit` (or `1` when uncalibrated) to the generator.

### Bug 2: Nothing generated regardless of spread value

**Problem:** After renaming the fields, `vegetation.ts` still referenced `layer.lakeSpreadPx` and `layer.riverSpreadPx` (now `undefined`) in four places in the per-pixel density loop and water-attenuation block. `lakeDist[i] < undefined` is always `false`, so density stayed 0 for every pixel.

**Fix:** All four occurrences updated to use the local `lakeSpreadPx`/`riverSpreadPx` variables computed at the top of `generateVegetation` from the renamed fields + `pixelsPerUnit`.

### Bug 3: Texture selector overflowed panel width

**Problem:** `SegmentedControl fullWidth` with 5 options ("Gradient", "Organic", "Stipple", "Hatch", "Cellular") was wider than the parameter panel; the last item was barely visible.

**Fix:** Replaced with a `Select` dropdown. Also added descriptive sub-labels in the dropdown options (e.g. "Organic (fBm noise)", "Stipple (Bayer dither)").

## Decisions Made

- **Spread stored in display units, converted at generation time** — avoids round-trip conversion issues in controlled inputs; makes the values meaningful to the user.
- **No spread upper cap** — removed arbitrary `max={500}`/`max={300}` caps; the BFS can handle any radius.
- **`dataUrl`/`generating` not persisted** — data URLs are large and regeneratable; always stripped from localStorage.
- **`addVegetationLayer` disabled until water features exist** — vegetation density depends on water distance fields; no point adding a layer before detection runs.
