## localStorage Persistence (Auto-Restore on Startup)

### Feature Overview

All app parameters are now persisted to `localStorage` and automatically restored on startup. The app relaunches exactly where the user left off â€” same contour settings, style, calibration, markers, frame config, zoom, and active tab. Files (heightmap + terrain image) are re-read from their saved paths and all derived data (hillshade, contours) is recalculated automatically.

Pressing **Reset** clears the localStorage key and resets all state to defaults.

### What Is Persisted

Via Zustand `persist` middleware with `partialize`:

- `heightmapPath`, `terrainImagePath` â€” file paths for auto-reload
- `parameters` (ContourParameters)
- `style` (ContourStyle)
- `hillshadeParams` (HillshadeParameters)
- `elevationCalibration` (ElevationCalibration)
- `activeTab`, `mapZoom`, `overlayOnly`, `overlayBrightness`
- `frame`, `title`, `compass`, `legend`, `measureBar`
- `elevationFlags`, `slopeArrows`, `ruggednessFlags`
- `ruggednessColorBySeverity`

### What Is NOT Persisted (Recalculated)

- `heightmap` (Float32Array binary â€” re-read from file path)
- `terrainImageUrl`, `hillshadeImageUrl` (blob URLs â€” regenerated after file read)
- `hillshadeDirty`, `contoursDirty`, `isDirty`, version counters, snapshots
- `mapTool` (reset to `'none'`)
- `fileLoadingMessage`, `hillshadeGenerating`, `contoursGenerating`

### Restore Sequence (`App.tsx`)

A one-time `useEffect` on mount reads `heightmapPath` / `terrainImagePath` from the already-hydrated store and runs:

1. **Terrain image first** (fast): calls `restoreTerrainImage(path, url)` â€” sets path + URL without forcing `activeTab: 'terrain'`
2. **Heightmap second**: calls `setHeightmap(path, info)` â€” increments `hillshadeVersion` and `contoursVersion`, triggering hillshade + contour regeneration via existing hooks

Loading terrain first ensures `setHeightmap` sees `terrainImageUrl` as truthy, so it preserves the persisted `activeTab` rather than overriding it to `'hillshade'`.

If either file is missing or unreadable, the stale path is cleared from the store silently.

### `reset()` Behavior

```typescript
reset: () => {
  localStorage.removeItem('topocrafter-state')
  set(initialState)
},
```

Removes the storage key first, then sets state to defaults. The persist middleware writes defaults back immediately on the next state change.

## Display Resolution Readout, Wheel Zoom, Click-Drag Pan

### Features Added

**Toolbar resolution display**
Live CSS pixel dimensions (e.g. `1024Ã—768`) shown to the right of the zoom percentage in the toolbar. Driven by the same `ResizeObserver` already tracking the inner map div for measure bars â€” no extra overhead. Updates in real time as the window is resized or zoom changes. Stored as `mapDisplaySize: { w, h } | null` in the store (not persisted).

**Mouse wheel zoom**
Wheel over the map area now adjusts the zoom slider instead of scrolling the container. Uses the same logarithmic step size as the slider's snap points (`Â±5` in logâ‚‚ space â‰ˆ 7% per notch). Implemented via a non-passive `wheel` event listener (required to call `preventDefault()`) on the scroll container, using a `mapZoomRef` to avoid re-registering the listener on every zoom change.

**Click-and-drag pan**
In default mode (no tool active), clicking and dragging the map area pans the scroll container â€” same effect as using the scrollbars. Cursor shows `grab` at rest, `grabbing` while dragging. Only activates when `mapTool === 'none'` and a map is loaded. Clicking on a flag/marker still works normally because `handleItemMouseDown` already calls `e.stopPropagation()`. SVG cursor changed from `'default'` to `'inherit'` so it picks up the parent's grab cursor.

## Swamp / Marsh Marker Tool + Per-Marker Boldness and Opacity

### Feature Overview

New **swamp-marker** annotation tool drops a 5-shoot marsh-grass icon. All four annotation types (elevation flags, slope arrows, ruggedness flags, swamp markers) now support per-instance **boldness** (Thin / Normal / Bold) and **opacity** captured at drop time from per-type defaults editable in the parameter panel.

### Swamp Marker Icon

Five SVG elements from a shared base point, scaled by `labelFontSize Ã— sizeFactor`:
- 3 upright `<line>` shoots: center (straight up), left (âˆ’22%), right (+22%)
- 2 `<path>` quadratic BÃ©zier bent shoots arcing outward and down

`sizeFactor` is randomized `0.75 + Math.random() * 0.5` at drop time, stored per-instance for stable rendering.

### Per-Instance Boldness / Opacity

**All four marker types** gained `boldness?: 1 | 2 | 3` and `opacity?: number` on their instance interfaces (`ElevationFlag`, `SlopeArrow`, `RuggednessFlag`, `SwampMarker`). At placement time these are copied from the per-type defaults in the store. Stroke multipliers: Thin = 0.6Ã—, Normal = 1.0Ã—, Bold = 1.8Ã— of base stroke width, applied via `bw(base, boldness)` helper in `MapCanvas`.

### Parameter Panel Controls

The **Markers & Annotations** collapsible section now has a subsection for each type:
- **Elevation Flags**: Stroke weight SegmentedControl + Opacity slider
- **Slope Arrows**: Stroke weight SegmentedControl + Opacity slider

- **Swamp Markers**: 3 quick-color swatches (blue `#1976D2`, green `#388E3C`, black `#212121`) + ColorInput picker + Stroke weight + Opacity

### Legend Integration

All four marker types appear as toggleable items in the Legend (Framing section). Swamp markers render in the canvas export legend as the same 5-shoot icon using `quadraticCurveTo`, color-matched to `swampMarkerColor`.
