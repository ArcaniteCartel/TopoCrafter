## Features Implemented This Session

### 1. Terrain / Hillshade Tab System

**User request:** Add Terrain and Hillshade tabs to the main view. Hillshade is always available once a heightmap is loaded (auto-generated on first load). Terrain tab only appears when a custom terrain image is loaded. Switching tabs changes which base image is shown beneath the overlay.

**Design decisions (Q&A):**
- Tab labels: "Terrain" and "Hillshade" (not "Heightmap")
- First heightmap load → auto-trigger hillshade generation immediately
- Dirty state after that → Recalculate button only (no auto-trigger)
- Hillshade controls in ParameterPanel are always visible when heightmap is loaded (not gated on `terrainIsHillshade`)

**Architecture change — `terrainIsHillshade` removed:**

Before: `terrainImageUrl` stored both the user's terrain image AND the auto-generated hillshade (whichever was "active"). `terrainIsHillshade: boolean` tracked which kind was displayed.

After:
- `terrainImageUrl` — user's custom terrain image only (null if none loaded)
- `hillshadeImageUrl` — auto-generated hillshade (null until first heightmap load completes)
- `activeTab: 'terrain' | 'hillshade'` — which tab/image MapCanvas renders
- `baseImageUrl` derived in MapCanvas: `activeTab === 'terrain' ? terrainImageUrl : hillshadeImageUrl`

**Auto-trigger on first load:**
`setHeightmap` now increments both `contoursVersion` and `hillshadeVersion`. `useHillshade` hook guard simplified to `if (!heightmap) return` (was `if (!heightmap || !terrainIsHillshade) return`). `setTerrainHillshade(url)` now writes to `hillshadeImageUrl` instead of `terrainImageUrl`.

**Tab UI:** Mantine `Tabs` in `App.tsx` above `MapCanvas`. Each tab appears only when its image is available. A `useEffect` in App.tsx corrects invalid active tab (e.g., terrain tab selected but terrain image unloaded). `setTerrainImage` sets `activeTab: 'terrain'`. `setHeightmap` sets `activeTab: 'hillshade'` if no terrain is loaded, otherwise keeps current tab.

**Files changed:**
- `types/index.ts`: Replaced `terrainIsHillshade` with `hillshadeImageUrl`, `activeTab`
- `store/useStore.ts`: Refactored `setTerrainImage`, `setTerrainHillshade`, `setHeightmap`; removed `setTerrainIsHillshade`; added `setActiveTab`; replaced all `state.terrainIsHillshade` checks with `!!state.heightmap`
- `hooks/useHillshade.ts`: Removed `terrainIsHillshade` subscription and guard
- `App.tsx`: Added `Tabs` component; subscribes to `heightmap`, `terrainImageUrl`, `activeTab`, `setActiveTab`
- `MapCanvas.tsx`: Subscribes to `hillshadeImageUrl`, `activeTab`; derives `baseImageUrl`; renders `baseImageUrl` for the `<img>` tag
- `ParameterPanel.tsx`: Replaced `terrainIsHillshade` gate with `!!heightmap` for hillshade controls section
- `FilePanel.tsx`: Removed `terrainIsHillshade` / `setTerrainIsHillshade` entirely; simplified Terrain Image section (no more "Hillshade" badge/text)

### 2. Clear Pending Changes

**User request:** A "Clear Pending Changes" button that restores settings to the last-rendered state. Active whenever `hillshadeDirty || contoursDirty`.

**Snapshot pattern:** Four snapshot fields added to `ProjectState`:
```ts
snapshotParams: ContourParameters | null
snapshotStyle: ContourStyle | null
snapshotHillshadeParams: HillshadeParameters | null
snapshotElevationCalibration: ElevationCalibration | null
```

Snapshot is updated whenever:
- `setHeightmap` is called (initial load triggers both renders; snapshot current settings)
- `triggerHillshade` is called (user explicitly recalculates)
- `triggerContours` is called (user explicitly recalculates)

`clearPendingChanges` action restores all four fields from snapshot and clears both dirty flags. Single snapshot covers both hillshade and contours — most changes dirty both, and the reference point is "last time anything was recalculated".

**FilePanel.tsx changes:**
- Added `clearPendingChanges` to destructured store values
- Added `hasPendingChanges = (hillshadeDirty || contoursDirty) && !!heightmap`
- Added "Clear Pending Changes" button (orange, subtle variant) below "Recalculate Map" in the same Paper
- Removed explicit `setHillshadeDirty(false)` / `setContoursDirty(false)` calls from `handleLoadHeightmap` (now handled by `setHeightmap` in store)

### 3. Export Dropdown

**User request:** Replace "Export Merged Image" button with an "Export" dropdown menu containing 5 items.

**Export items:**

| Item | Base image | Contours | Annotations | Opacity |
|---|---|---|---|---|
| Export merged Terrain image | `terrainImageUrl` | ✓ | ✓ | 1.0 |
| Export merged Hillshade image | `hillshadeImageUrl` | ✓ | ✓ | 1.0 |
| Export overlay layer image | none (transparent) | ✓ | ✓ | 1.0 |
| Export unmarked Hillshade | `hillshadeImageUrl` | ✗ | ✗ | — |
| Export visible map | active tab's image | ✓ | ✓ | `style.opacity` |

"Export visible map" uses current opacity, respects the active tab, and is designed for future per-layer visibility toggles.

**`utils/export.ts` (new file):**
```ts
interface ExportLayerConfig {
  baseImageUrl: string | null
  includeContours: boolean
  includeAnnotations: boolean
  contourOpacity: number
}

async function exportToBlob(config: ExportLayerConfig): Promise<Blob>
```

Canvas compositing approach:
1. Get SVG elements by ID: `document.getElementById('contour-svg')` / `'annotation-svg'`
2. Measure displayed dimensions via `getBoundingClientRect()` → canvas size
3. Draw base image (stretched to fill canvas)
4. Draw contour SVG at given opacity (SVG cloned with explicit `width`/`height` attributes to fix serialization)
5. Draw annotation SVG at full opacity
6. Return `canvas.toBlob('image/png')`

SVG serialization: `new XMLSerializer().serializeToString(clone)` → `data:image/svg+xml;charset=utf-8,...` data URL. Clone sets explicit pixel dimensions because `width="100%"` doesn't render correctly when an SVG is used as an image source.

**Export flow in `Toolbar.tsx`:**
- `handleExport(type)` → calls `exportToBlob(config)` → `window.electronAPI.saveFile(...)` → `window.electronAPI.writeFile(path, new Uint8Array(blob.arrayBuffer()))`
- Inline error display (small red text) replaces console.error only

**Per-item disable logic:**
- "Export merged Terrain image": disabled when no `terrainImageUrl`
- "Export merged Hillshade image": disabled when no `hillshadeImageUrl`
- "Export overlay layer image": disabled when no `heightmap`
- "Export unmarked Hillshade": disabled when no `hillshadeImageUrl`
- "Export visible map": disabled when no `baseImageUrl` for active tab

**SVG IDs added in `MapCanvas.tsx`:**
- `id="contour-svg"` on the contour SVG element
- `id="annotation-svg"` on the annotation SVG element (the `flagSvgRef` element)
