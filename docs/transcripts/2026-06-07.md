## Building Legend Entries

### Feature Overview

Added a `showBuildings` toggle to the legend parameters. When enabled, the legend shows one entry per unique `(templateId, color)` combination found on the map. Entries whose user-facing labels are identical are merged into a single legend row showing multiple shape icons side by side with one description label.

Labels default to the full catalog template name; each can be overridden per `(templateId, color)` key in the parameter panel.

### Legend Config Changes

Added to `LegendConfig` in `types/index.ts`:

```typescript
showBuildings: boolean          // default: true
buildingLabels: Record<string, string>  // key: `${templateId}::${color}`
```

Added `templateId: string` to `BuildingEntry` so legend grouping has a stable key per placed building.

### Legend Overlay (LegendOverlay in MapCanvas.tsx)

`LegendOverlay` receives a new `buildings: BuildingEntry[]` prop. After the static items array is built, a dynamic block iterates placed buildings and:

1. Collects unique `(templateId, color)` pairs into a `seen` map
2. Groups entries that share the same label string into a `byLabel` map
3. Pushes one `{ type: 'building', label, buildingShapes: [...] }` item per label group

`buildingShapes` carries `{ shape, color, widthM, depthM }` for each entry in the group.

### Relative Scale Between Building Icons

All building shape icons in the legend share a single `buildingGlobalScale` derived from the real-world dimensions of all placed buildings. The largest building fills its slot; all others are drawn proportionally smaller.

Implementation: the `items.map` render loop is wrapped in an IIFE that computes `buildingGlobalScale` before the loop begins:

```typescript
{(() => {
  const bBh = rowH * 0.72
  let buildingGlobalScale = 1
  const allBuildingShapes = items.flatMap(it => it.buildingShapes ?? [])
  if (allBuildingShapes.length > 0) {
    let gs = Infinity
    for (const { widthM, depthM } of allBuildingShapes) {
      gs = Math.min(gs, sampW * 0.88 / widthM, bBh / depthM)
    }
    buildingGlobalScale = isFinite(gs) ? gs : 1
  }
  return items.map(({ type, label, color, buildingShapes }, i) => {
    // ...
    // inside building sample render:
    const pw = widthM * buildingGlobalScale
    const pd = depthM * buildingGlobalScale
    const d = buildingPath(shape, cx, midY, pw, pd)
  })
})()}
```

## Points of Interest Tool

### Feature Overview

Added a `'poi'` map tool with three marker types for special geographic features:

- **Mine Entrance** — USGS-style crossed pick-and-shovel symbol (X with perpendicular end-cap ticks at all four shaft tips)
- **Bridge** — two bold parallel line segments; user controls color, length, separation, stroke weight, rotation
- **Cave Entrance** — capital Greek Ω character; user controls color, symbol size, font family

All types support: optional per-marker map label (place name) with color/size/font; a per-type "Color Significance Label" that overrides the standard legend label; one legend entry per type present on the map.

### Type System (`types/index.ts`)

```typescript
export type PoiType = 'mine' | 'bridge' | 'cave'

export interface PoiEntry {
  id: string; x: number; y: number; type: PoiType; color: string
  mineSize?: number          // meters
  bridgeLength?: number; bridgeSeparation?: number  // meters
  bridgeStrokeWeight?: number  // px
  bridgeRotation?: number    // degrees
  caveSize?: number          // meters
  caveFontFamily?: string
  label?: string; labelColor?: string; labelSizeM?: number; labelFontFamily?: string
}

export interface PoiDefaults { /* per-type color/size defaults + legend override labels */ }
export const defaultPoiDefaults: PoiDefaults = { ... }
```

`showPois: boolean` added to `LegendConfig`. `pois`, `poisVisible`, `poiDefaults`, `selectedPoiId` added to `ProjectState`.

### Store (`useStore.ts`)

Actions: `addPoi`, `updatePoi`, `removePoi`, `setPoisVisible`, `updatePoiDefaults`, `setSelectedPoiId`. `removePoi` auto-clears `selectedPoiId` if the removed POI was selected.

### Map Canvas (`MapCanvas.tsx`)

Module-level SVG helper components:
- `PoiMineSymbol` — two shaft lines + four perpendicular tick marks
- `PoiBridgeSymbol` — two parallel lines with rotation transform
- `renderPoiSymbol(poi, pixelsPerMeter)` — dispatches to the appropriate symbol or Ω text

Key interaction details:
- **Hit area**: each POI `<g>` has a `<circle fill="transparent">` as its first child (essential — stroke-only SVG elements have no pointer-event hit area)
- **Selection ring**: double-ring `rgba(0,0,0,0.55)` solid outer + white dashed inner, both `vectorEffect="non-scaling-stroke"`, visible on any background
- Drag, Delete key, Escape all work as with other marker types
- `poiDefaultsRef` provides stable closure for placement handler
- `setSelectedPoiId` called in all deselection paths alongside `setSelectedItem(null)`
- Placed markers rendered after buildings; hover preview shown when `mapTool === 'poi'`
- LegendOverlay extended with `pois` and `poiDefaults` props; one legend row per type present, using significance label override or standard name

### Parameter Panel (`ParameterPanel.tsx`)

**UX: selected-first layout** — When a POI is selected, the panel shows a blue-bordered "Editing: [Type]" box as the first control in the section (above all defaults). The box contains all per-marker controls (color, type-specific geometry, map label, delete button). When nothing is selected, the "New Marker Defaults" section appears instead.

This prevents the confusion where users changed the *defaults* size thinking they were editing the selected marker.

Subscriptions added: `removePoi`, `setSelectedPoiId`.
