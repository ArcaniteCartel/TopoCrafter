## Cross-Platform Builds from Windows

**Question:** Can the app be built for macOS/Linux from a Windows machine?

**Answer:** Generally no for production builds:

- **macOS** — cannot be cross-compiled from Windows at all. Apple's code-signing toolchain
  only runs on macOS, and DMG/PKG packaging requires it even without signing.
- **Linux** — theoretically possible in some cases but unreliable in practice.

The standard approach is a CI service (e.g. GitHub Actions) with separate runners per
platform: Windows runner for `.exe`, macOS runner for `.dmg`, Linux runner for
`.AppImage`/`.deb`. electron-builder is designed with this workflow in mind.

## Hex Grid Variants — Pointy-Top and Rotated 45°

Added two new hex grid options to the overlay export modal alongside the
existing flat-top hex:

- **Hex (pointy-top)** — vertices oriented vertically (perpendicular to map width); 30°
  rotation from flat-top.
- **Hex (rotated 45°)** — 45° rotation from flat-top; no edges horizontal or vertical.

**Implementation:** Replaced the ad-hoc flat-top `drawHexGrid` with a generalised version
that accepts a rotation angle in radians. The Bravais lattice basis vectors
`b1 = (3R/2, R√3/2)` and `b2 = (0, R√3)` are rotated by the angle, then used to iterate
integer (n, m) combinations centred on the canvas. Culling skips any hex centre outside the
canvas bounds. `OverlayGridType` expanded from `'square'|'hex'` to
`'square'|'hex-flat'|'hex-pointy'|'hex-rotated'`.

## Live Cursor Readout for Elevation and Slope Tools

While a map tool is active, hovering the map shows a live preview next to the crosshair:

- **Elevation flag tool** — elevation value rendered to the upper-right of the cursor at
  75% opacity, in the same font/style as placed flag labels.
- **Slope arrow tool** — full ghost arrow + slope degree label centred on the cursor at
  75% opacity, identical geometry to placed arrows.

Preview disappears on mouse leave, tool deactivation, or Escape. Suppressed while an
existing marker is being dragged (`dragPos !== null`).

**Implementation** (`src/renderer/src/components/MapCanvas/MapCanvas.tsx`):
- Added `hoverPos: DragPos | null` state.
- `handleSvgMouseMove` computes elevation/slope at cursor via existing `computeElevationAt`
  / `computeSlopeAt` callbacks and updates `hoverPos`. Guard: `dragRef.current` prevents
  updates during drags.
- `handleSvgMouseLeave` clears `hoverPos`.
- `useEffect([mapTool])` clears `hoverPos` when tool is deactivated.
- Preview rendered inside annotation SVG after placed markers (renders on top), using an
  IIFE to keep conditional geometry inline without extracting a helper.

## Overlay-Only Toggle

Added an "Overlay only" switch to the Hillshade section header. When on:
- The base image (terrain or hillshade) is hidden using `visibility: hidden` (keeps layout
  dimensions so SVG overlays remain correctly sized).
- The inner zoom div receives a light grey background derived from `overlayBrightness`.
- All hillshade sliders except Brightness are disabled.
- The Brightness slider reads/writes a separate `overlayBrightness` store value — changing
  it does **not** set `hillshadeDirty`. Normal hillshade brightness is preserved
  independently, so toggling back and forth restores each mode's own brightness level.

**Store additions:** `overlayOnly: boolean`, `overlayBrightness: number` (default 0.65),
`setOverlayOnly`, `setOverlayBrightness`.

**Background colour formula:** `rgb(N, N, N)` where `N = round(255 × (0.85 + 0.15 × overlayBrightness))` — ranges from 217 at brightness 0 to 255 (white) at brightness 1; near-white (~248) at the default 0.65.

## Framing Feature — Design Discussion

Agreed on a "Framing" system to add a composed border area around the map for export.
This is a phased build.

### Layout (inside → outside)
```
┌──────────────────────────────────────────┐
│  Title (top-left, above measure bar)     │
│  Top measure bar (longitude or latitude) │
│  ┌────────────────────────────────────┐  │
│  │            BORDER                  │  │
│  L  measure  │   MAP + OVERLAY  │  R  │  │
│  e    bar    │                  │  m  │  │
│  f           │                  │  e  │  │
│  t           │                  │  a  │  │
│              │            BORDER│     │  │
│  └────────────────────────────────────┘  │
│  Bottom measure bar                      │
│  [gap] [Compass - centre] [Legend - right] [gap] │
└──────────────────────────────────────────┘
```

### Planned phases
1. **Frame shell** — toggle, margin (size + colour), border (width + colour + fancy styles),
   live preview in MapCanvas showing the full composition
2. **Title** — text, font, size, colour, bold/italic
3. **Compass rose** — N/S/E/W labels with style controls, per-arm arrowhead toggle
4. **Legend** — auto-derived from overlay parameters (contour styles, flag/arrow markers)
5. **Measure bars + geolocation** — 2D world map picker (Mercator/Robinson toggle),
   planet size relative to Earth, lat/lon in DMS, derived tick spacing along frame edges

### Key design decisions
- Frame visible in live preview (map shrinks inside canvas to accommodate frame)
- Element positions fixed: title top-left (above top measure bar), legend bottom-right,
  compass rose bottom-centre
- 2D world map with draggable pin for lat/lon selection (not 3D globe)
- Mercator and Robinson projections, toggle between them
- Bundled simplified Natural Earth GeoJSON coastline data
- Compass orientation handled via label content (user sets which arm label = N) rather
  than rotation
