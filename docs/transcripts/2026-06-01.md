## build:win — Still Needed?

**Question:** Is `npm run build:win` still needed given the dev workflow?

**Answer:** Not for day-to-day development — `npm run dev` is all that's needed. `build:win`
remains in `package.json` and is functional (the winCodeSign + NSIS issues were resolved in
the 2026-05-31 session). It would only be used when producing a distributable `.exe`
installer to share or deploy the app on another machine.

## Cross-Platform Builds from Windows

**Question:** Can the app be built for macOS/Linux from a Windows machine?

**Answer:** Generally no for production builds:

- **macOS** — cannot be cross-compiled from Windows at all. Apple's code-signing toolchain
  only runs on macOS, and DMG/PKG packaging requires it even without signing.
- **Linux** — theoretically possible in some cases but unreliable in practice.

The standard approach is a CI service (e.g. GitHub Actions) with separate runners per
platform: Windows runner for `.exe`, macOS runner for `.dmg`, Linux runner for
`.AppImage`/`.deb`. electron-builder is designed with this workflow in mind.
