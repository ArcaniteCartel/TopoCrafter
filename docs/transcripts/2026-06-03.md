## Positioning System — Shared 12-Position `FramePosition` Type

Replaced the fixed positioning for Title, Compass, and Legend with a shared 12-value enum covering all frame margin zones. Also added multi-column support to the Legend.

### Design

The 12 positions cover:

| Zone | Positions |
|------|-----------|
| Top margin strip (full width) | `top-left`, `top-center`, `top-right` |
| Right margin strip (map height only) | `right-top`, `right-middle`, `right-bottom` |
| Bottom margin strip (full width) | `bottom-right`, `bottom-center`, `bottom-left` |
| Left margin strip (map height only) | `left-bottom`, `left-middle`, `left-top` |

**Key clarification:** The top/bottom margin corners (`top-left`, `top-right`, `bottom-left`, `bottom-right`) span the full width of the composition including the side margin areas. The side positions (`right-top`, `left-top`, etc.) are within the side strips and start/end at the map edge, not at the composition corner — so corners are vertically above/below the side near-top/near-bottom positions.

### Types (`src/renderer/src/types/index.ts`)

```typescript
export type FramePosition =
  | 'top-left'    | 'top-center'    | 'top-right'
  | 'right-top'   | 'right-middle'  | 'right-bottom'
  | 'bottom-right'| 'bottom-center' | 'bottom-left'
  | 'left-bottom' | 'left-middle'   | 'left-top'
```

- `LegendPosition` type removed (replaced by `FramePosition`).
- `TitleConfig`: added `position: FramePosition` (default: `'top-left'`).
- `CompassConfig`: added `position: FramePosition` (default: `'bottom-center'`).

### MapCanvas — positioning helpers

Two shared helper functions added before `LegendOverlay`:

**`getElementPositionStyle(pos, frame, w, h, edgeGap): React.CSSProperties`**
For SVG elements with known pixel dimensions (compass, legend). Returns `position: absolute` CSS that centers the element at the position's anchor:
- Top/bottom strip positions: uses `top`/`bottom` with pixel offset (`mt/2 - h/2`), horizontal via `left`/`right` or `left: 50%` + `translateX(-50%)`.
- Side positions: uses `mr/2 - w/2` / `ml/2 - w/2` for horizontal centering in the strip; `top: calc(50% + mco)` + `translateY(-50%)` for middle (where `mco = (mt - mb) / 2` corrects for unequal top/bottom margins).

**`getTitleWrapperStyle(pos, frame, edgeGap): React.CSSProperties`**
For the title text div (variable width). Returns `position: absolute` CSS that places the wrapper's center at the anchor. Uses `translate(-50%, -50%)` etc. on the wrapper; the inner `<div>` carries `rotate(-90deg)` (left strip) or `rotate(90deg)` (right strip) for vertical text.

### Legend — multi-column layout

Items are laid out in column-major order: column 0 fills completely before column 1 starts.

```typescript
const cols = Math.max(1, Math.min(legend.columns, items.length))
const rows = Math.ceil(items.length / cols)

// Item i → column = floor(i / rows), row = i % rows
const col = Math.floor(i / rows)
const row = i % rows
const sx1 = pad + col * (colW + colGap)
const midY = pad + row * rowH + rowH / 2
```

Box dimensions scale accordingly:
```typescript
const colW = sampW + gapX + maxLabelLen * fs * 0.56
const boxW = pad + cols * colW + (cols - 1) * colGap + pad
const boxH = pad + rows * rowH + pad
```

The same column-major math is replicated in `drawLegend` in `export.ts` using `ctx.measureText()` for exact label widths.

### export.ts — canvas position helpers

Two helper functions added before `drawFrameBorder`:

**`getPositionCenter(pos, frame, totalW, totalH, edgeGap): [number, number]`**
Returns the (cx, cy) center point for symmetric elements (compass). Side-strip positions use the center of the map area vertically: `mt + mapH/2`.

**`getBoxOrigin(pos, frame, totalW, totalH, boxW, boxH, edgeGap): [number, number]`**
Returns top-left (x, y) for rectangular elements (legend). Corner positions (top-left, top-right, bottom-left, bottom-right) left/right-align the box at `edgeGap`. Side positions center the box horizontally within the strip and align the near edge to the margin boundary.

**`drawTitle`** now accepts `totalW, totalH`; uses `getPositionCenter` to find the anchor, then:
- For left/right strips: `ctx.save(); ctx.translate(cx, cy); ctx.rotate(±π/2); ctx.textAlign = 'center'; ctx.fillText(text, 0, 0); ctx.restore()`
- For top/bottom strips: `ctx.textAlign` set to `left`/`center`/`right` based on the position variant

**`drawCompassRose`** call sites updated to compute `(cx, cy)` via `getPositionCenter(compass.position, ...)` instead of hardcoded `(totalW/2, totalH - marginBottom/2)`.

**`drawLegend`** updated with multi-column layout and `getBoxOrigin` for positioning.
